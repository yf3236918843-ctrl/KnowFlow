# EasyLearn

EasyLearn 不是一个"替你讲课"的 AI。

它更像一个会在做题过程中逐渐理解学生的私人教师：
记住你最近学了什么、卡在哪、喜欢怎样的讲法，
然后把这种理解用到下一次教学里。

目标不是回答更多题，而是让 AI 对
"这个学生是谁、现在处于什么状态、该怎么教他"
越来越有感觉。

---

## 它和普通的 AI 教育产品有什么不同？

大多数 AI 教学工具的重点是"把这道题讲明白"。

EasyLearn 的重点是另一件事：
在讲题的同时，持续积累对学生的理解。

它区分三类信息：

- **学生主动确认的偏好**：比如"不要直接给答案"。
  确认后就是规则，教师必须遵守。

- **做题过程中暴露的状态**：比如"这道题做对了，但过程卡在 ε-δ"。
  系统自动记录，不需要学生自己记。

- **对当前学生有效的教学策略**：比如"用图形解释效果更好"。
  这些信息积累在跨对话记忆中，影响下一轮教学。

这些信息不会只是存档。它们会反过来影响下一次做题时的教学。

---

## 现在能做什么

- **顺序做题**：按题库顺序进入对话式解题
- **过程记录**：每道题的结果自动记录（对错、掌握度、薄弱点）
- **错题收录**：值得回看的内容，自动收录到薄弱池
- **偏好系统**：偏好的生成、确认、修改，后续自动注入
- **会话级记忆**：最近几次的教学状态自动保留，
  不用每次重新说明情况
- **流式交互**：思考过程、卡片生成、工具调用尽量可见

---

## 一个典型场景

今天你说："这题我完全不会。"

AI 不只是解释这道题。

它同时在做的事：
- 记录这道题你做错了，归入错题池
- 注意到你对某个概念反复卡住
- 捕捉到你更适合哪种讲解方式
- 在后台问自己：这些要不要进入偏好，等你确认？

下次你回来，它知道你上次做到哪、卡在哪。
不用你重新说。

---

## 使用流程

1. 导入你的题（教材、习题册、真题卷）
2. 开始做题，每道题在对话中完成
3. 做完了：系统自动记录结果，归类到三池
4. 复习时：系统按池出题，不需要你规划
