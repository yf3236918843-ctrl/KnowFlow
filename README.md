# EasyLearn

EasyLearn 不是一个“替你讲题”的 AI。

它更像一个会在做题过程中逐渐理解学生的私人教师：
记住你最近在学什么、卡在哪、偏好怎样的讲法，
再把这些理解继续用到下一次教学里。

它的目标不是单次回答得更漂亮，
而是让 AI 对“这个学生是谁、现在处于什么状态、接下来该怎么教”越来越有感觉。

---

## 为什么做这个项目？

很多 AI 教学产品关注的是：

“这道题怎么讲清楚？”

EasyLearn 更关心另一件事：

“在讲这道题的同时，能不能逐渐形成对这个学生的理解？”

真实的老师不会每次都像第一次见到学生一样。
他会慢慢知道：

- 这个学生最近在学什么
- 哪些内容只是接触过，不能默认已经掌握
- 他更适合被怎样引导
- 他会反复卡在哪些地方
- 哪些要求是他明确表达过、之后都应该遵守的

EasyLearn 想做的，就是把这种“越来越懂学生”的能力做出来。

---

## 它和普通 AI 做题助手有什么不同？

大多数 AI 做题助手的重点，是把当前这轮对话做好。

EasyLearn 除了处理当前题目，还会持续沉淀三类信息：

- 学生主动确认的强偏好  
  例如：“不要直接给答案”、“讲慢一点”、“先讲思路再讲公式”

- 会话里暴露出的学习状态  
  例如：“最近在学极限”、“洛必达法则还不稳”、“这类题会做但一变式就慌”

- 对这个学生有效的教学方式  
  例如：“图形解释比抽象定义更有效”、“先让他判断题型再提示工具更顺”

这些信息不会只是存档，而会反过来影响之后的教学。

也就是说，EasyLearn 关注的不是“把一题讲完”，
而是“在一次次做题中形成教师对学生的认知”。

---

## 当前能做什么？

- 顺序做题  
  围绕题库按顺序进入对话式解题流程

- 流式交互  
  尽量把思考过程、结构化卡片、工具过程可见化，而不是整段同步返回

- 错题 / 薄弱点收录  
  在做题过程中把值得回看的内容沉淀下来

- 偏好系统  
  支持偏好的生成、确认、修改，并在后续场景中注入生效

- 会话级总结  
  每轮题目完成后生成本轮学习记录，沉淀“这次学到了什么、暴露了什么问题”

- 会话级教师记忆注入  
  在后续教学中带入最近几轮学习状态，让模型不再把学生当成第一次见面的人

- 题库管理  
  支持项目 / 题库 / 题组 / 题目组织，以及 JSON 形式导入题目

## 一个典型场景

今天你说：

> “老师，这题我完全不会。”

EasyLearn 不只是回答这道题。

它还会逐渐意识到：

- 你最近正在学哪一块
- 你没学过什么，不能默认你懂
- 你更吃哪种讲法
- 你是不是在重复暴露同一种卡点
- 这轮对话里哪些信息值得进入偏好系统

下次再开始时，它不应该像第一次认识你。

这就是这个项目真正想做的事。

---

## 设计原则

- 学生自己决定学什么  
  系统不替用户规划人生，只围绕用户已有材料和目标工作

- 做题过程比标准答案更重要  
  重点不是“答对没”，而是“是怎么卡住、怎么被引导出来的”

- 偏好是可编辑的  
  学生对教学方式的要求，应该能被记录、修改、覆盖，而不是写死在提示词里

- 记忆服务于教学  
  记忆不是为了“像聊天机器人一样记住一切”，而是为了更像一个真正的老师

- 用户体验优先  
  尽量流式、尽量可见、尽量少黑盒

## 当前状态

项目仍在持续迭代中，但核心方向已经明确：

- 以“顺序做题”为主线
- 以“偏好系统 + 会话级记忆”为核心能力
- 逐步把 AI 从“单轮答题助手”推进成“越来越懂你的老师”

目前重点还在把主流程打磨稳定，把记忆与偏好系统调顺。

## 这个项目适合谁？

如果你也在关注这些问题，这个项目可能对你有价值：

- AI 能不能不只会答题，而是真的逐渐理解学生？
- 教学型 Agent 的“记忆”到底应该记什么？
- 偏好系统如何真正参与后续教学，而不是只做展示？
- 如何把题目练习、偏好积累、会话记忆做成同一个闭环？
