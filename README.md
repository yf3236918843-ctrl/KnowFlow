# 简学 / EasyLearn

学习是一件很个人的事。每个人的理解方式、接收信息的节奏、容易卡住的地方都不一样。大多数 AI 教育工具试图扮演"老师"的角色——替你规划内容、替你讲解、替你决定学什么。这不适合所有人。

简学不是老师。它做的事情是：把你做题的过程记录下来，从中理解你的偏好和学习状态，然后用这些理解帮你更好地复习。

---

## 使用流程

### 开始

你有一套自己的学习材料——教材、习题册、真题卷。这些是你自己选的，你知道自己要学什么。简学不需要替你决定学什么，你把自己的题放进去就好。

### 做题

每道题你会在对话中完成。AI 会引导你思考，但不会直接给答案。做完后系统会记录：这道题做对了还是做错了、你的掌握程度、暴露了哪些薄弱点。

如果你遇到一道觉得值得记下来的题，可以主动收录到错题本。AI 先生成草稿，你按自己的语言修改，最后确定。这个过程和做题在同一轮对话里完成。

每道题做完后系统会做一次总结——写入做题记录（不可改），同时在后台生成偏好候选（可改）。一次 LLM 调用，两份输出，分开处理。

### 沉淀

做题过程中你可能会说"讲太快了"、"别直接给答案"之类的话。一次这样的反馈不算什么，但如果你多次表达同样的倾向，系统会注意到，然后问你：要不要把这个记成偏好？

做题记录是事实，写入后不可改。偏好是意愿，随时可以调整。

每道题做完后自动归入三个池之一：
- **consolidate**：做对了，作为基础储备
- **mistake**：做错了，系统会分析错因
- **weakness**：你主动收录的，需要重点覆盖

### 复习

复习时系统从三个池出题。mistake 池的题分析上次错在哪，weakness 池的题出变式精准覆盖，consolidate 池的题作为安全底座混入。你不需要自己规划复习什么，系统会根据你积累的数据来安排。

## 设计思路

### 偏好即记忆

不存在独立的"记忆系统"。所有"系统知道关于这个学生的事"——偏好、习惯、做题记录、薄弱点——都归偏好系统管。不是刻意不做记忆引擎，是没必要——偏好已经够用了。

### 一起生成，分开处理

对话结束时，同一个 LLM 调用同时输出两份东西：做题记录（直接写入，不可逆）和偏好候选（进入确认循环，可逆）。不是分两阶段先后做，是同一段上下文一起输出。

### 弱信号先积累再确认

一次随口的抱怨可能是情绪，同类反馈出现三次才是模式。积累到阈值才触发确认，减少打扰。

### 不是答疑机器人

AI 不会主动给你讲课。它的职责是引导你思考、帮你把做题过程数字化、基于这些数据出复习题。你不需要被教，你只是需要有人帮你把做过的题管好。

## 架构

```
Pipeline
│
├── strategies/           ← 业务场景
├── SessionManager        ← LLM 会话管理
├── PreferenceEngine      ← 偏好体系
├── PromptManager         ← 提示词模板
└── Store                 ← 统一存储
```

| 层 | 模块 | 职责 |
|---|---|---|
| 0 | Log | 日志 |
| 1 | Store | 存储 CRUD |
| 2 | LLMGateway | LLM 调用、会话生命周期 |
| 3 | PromptManager | 模板管理 |
| 4 | PreferenceEngine | 偏好提取 / 积累 / 应用 |
| 5 | Pipeline | 流程编排 |
