# 知流

知流 想做的，不只是一个会讲题的 AI。

它更像一个会在长期使用中逐渐理解学生、
并进一步接管复习组织工作的私人教师系统。

它一边陪你做题，一边理解你最近学到了什么、卡在哪里、适合怎样的讲法；
再基于这些信息，决定你接下来该复习什么、什么时候复习、出什么题最合适。

目标不是把某一道题讲明白，
而是让学生尽量只专注于学习本身，进入持续做题和持续推进的心流。

---

## 这个项目想解决什么问题？

现实里的学习，真正消耗人的往往不只是“不会做题”。

而是这些额外负担：

- 错题要不要抄？
- 哪些题值得复习？
- 什么时候该回头复习？
- 复习时该看旧题，还是做变式题？
- 自己到底是不会，还是只是暂时没想起来？
- 最近学过什么，哪些已经会了，哪些只是“看过”但还没掌握？

直接使用AI工具只能回答“这题怎么做”。

知流 想进一步解决的是：

**能不能把做题过程本身数据化，进而自动组织后续复习，让学生尽量不用管理这些外围事务？**

---

## 直接使用AI与这个项目有什么不同？

直接使用 ChatGPT、Claude 这类通用 AI 工具，本身已经能把单道题讲得很清楚了。

知流 并不试图在“讲题能力”上取代或超越它们，而是想探索另一个更长的循环：

1. 学生做题
2. 系统记录过程与结果
3. 系统逐渐理解这个学生
4. 系统基于这些理解组织后续复习
5. 学生继续做题
6. 系统继续更新认知

也就是说，它不是单轮答题工具，而是一个面向长期学习过程的系统。

它既想回答当前问题，也想回答：

- 这个学生最近处于什么状态？
- 什么东西他已经真正掌握了？
- 什么只是接触过，还不能默认会？
- 什么讲法对他更有效？
- 下一轮最值得练的题是什么？
- 现在该不该复习？该复习哪一类？

## 核心价值

### 1. 越来越懂学生

系统会持续沉淀三类信息：

- 学生主动确认的强偏好  
  例如：“不要直接给答案”、“讲慢一点”、“不要默认我学过这个”

- 会话中暴露出的学习状态  
  例如：“最近在学极限”、“洛必达法则接触过但不稳”、“一变式就慌”

- 对这个学生有效的教学方式  
  例如：“图形解释更有效”、“先判断题型再提示工具更顺”

这些信息会继续进入后续教学上下文，让 AI 不再把学生当成第一次见面的人。

### 2. 让复习组织自动化

知流 的未来重点不只是讲解题目，
而是把做题过程和结果转成可用于复习调度的数据。

当系统知道：

- 你做对了还是做错了
- 是独立做出来的，还是提示后完成的
- 错在概念、方法、计算还是理解偏差
- 哪类题已经稳了，哪类题只是短暂会了
- 哪些内容已经隔了一段时间没碰

它就可以进一步做很多事：

- 生成更恰当的复习题
- 安排更合适的复习时机
- 混合旧题、错题、薄弱点题和变式题
- 减少无效重复
- 尽量把学生留在“继续做下一题”的心流里

理想状态下，学生不需要再额外管理：

- 纸质错题本
- 复习顺序
- 复习提醒
- 哪些题该重做
- 哪些知识点该回头补

系统来接手这些外围管理工作。

## 当前能做什么？

- 顺序做题  
  围绕题库按顺序进入对话式解题流程

- 流式交互  
  尽量把思考过程、结构化卡片、工具过程可见化

- 错题 / 薄弱点收录  
  把值得回看的内容沉淀下来，而不是散落在对话里

- 偏好系统  
  支持偏好的生成、确认、修改，并在后续场景中注入生效

- 会话级总结  
  每轮题目结束后生成这轮学习记录，沉淀学生状态与教学信号

- 会话级记忆注入  
  在后续教学中带入最近几轮学习状态，让模型保持连续认知

- 题库管理  
  支持项目 / 题库 / 题组 / 题目组织，以及 JSON 形式导入题目

## 一个典型场景

今天你说：

> “这题我完全不会。”

简学 不只是解释这道题。

它还会逐渐意识到：

- 你最近正在学什么
- 你没学过什么，不能默认你懂
- 你更适合哪种讲法
- 你是不是在重复暴露同一种卡点
- 这轮对话里哪些内容值得沉淀为偏好或学习记录

再往后，它不只会“记住这些”，
还会基于这些信息决定：

- 你接下来该练什么题
- 哪类题现在该回头复习
- 是继续推进新内容，还是先巩固旧内容

这才是它真正想做的事。

## 设计原则

- 学生自己决定学什么  
  系统不替用户决定目标，只围绕用户已有材料和学习过程工作

- 做题过程比单次答案更重要  
  重点不是“这题对没对”，而是“你是怎么做出来的、卡在哪里”

- 偏好是可编辑的  
  学生对教学方式的要求，应该能被记录、修改、覆盖

- 记忆服务于教学  
  记忆不是为了“什么都记住”，而是为了更像一个真实老师

- 复习组织应该自动化  
  学生应该把精力留给学习本身，而不是管理复习事务

- 用户体验优先  
  尽量流式、尽量可见、尽量少黑盒

## 当前状态

项目仍在持续迭代中，但方向已经明确：

- 以“顺序做题”为当前主线
- 以“偏好系统 + 会话级记忆”为核心能力
- 未来逐步扩展到更完整的自动复习系统

当前重点，是先把主流程打磨稳定，
再让“越来越懂学生”和“自动组织复习”真正形成闭环。
