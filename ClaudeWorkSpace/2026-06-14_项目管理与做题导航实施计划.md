# 项目管理与做题导航实施计划

## 目标

第一期先打通三件事：

1. 项目管理可用
2. JSON 题目追加导入可用
3. 顺序做题界面的题目路径与题目导航可用

## 实施顺序

### 1. 后端能力补齐

- Repo 增加 `project/bank/group/question` 的更新能力
- Repo 增加题组内题目追加能力
- Repo 增加题目删除后的顺序重排能力
- Repo 增加“题目路径解析”和“题目导航摘要数据”能力
- CRUD pipeline 暴露这些能力给前端

### 2. 回归测试先行

- 新增回归测试，覆盖：
  - 题组题目 JSON 追加后顺序正确
  - 删除单题后题号自动重排
  - 项目/题库/题组/题目内容可更新
  - 题目导航可返回 summary 与 session 关联数据

### 3. 前端壳层

- 去掉 `main.html` 自动登录
- 新增登录页
- 接入 `/api/me`、`/api/auth/login`、`/api/auth/register`、`/api/auth/logout`
- 侧边栏显示当前用户与退出入口

### 4. 项目管理视图

- 功能区新增“项目管理”
- 单列逐层钻取：
  - 项目列表
  - 题库列表
  - 题组列表
  - 题目列表
- 每层支持：
  - 新建
  - 重命名
  - 删除
- 题目层支持：
  - 编辑题目内容
  - 删除单题
- 题组层支持：
  - JSON 追加导入
  - 图片导入占位按钮

### 5. 顺序做题增强

- 左上显示 `项目 / 题库 / 题组 / 题号`
- 右上增加“题目导航”
- 导航内：
  - 未总结题：点击跳转/恢复
  - 已总结题：点击显示总结弹层

## 本轮验证

- Python 回归测试
- JS 语法检查
- 本地启动服务做一次手动 smoke test
